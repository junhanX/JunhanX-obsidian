mutex锁的特点：

抢不到锁就会睡眠让出cpu
只能在进程上下文使用
持锁期间允许调度


睡眠后 唤醒机制 

mutex 中有等待队列。mutex_unlock 负责唤醒。 事件驱动

```
Task B

mutex_lock()
      │
      ▼
锁被占用
      │
      ▼
加入 wait_list
      │
      ▼
TASK_UNINTERRUPTIBLE
      │
      ▼
schedule()
      │
      │（睡眠）
      │
Task A
mutex_unlock()
      │
      ▼
wake_up_process(Task B)
      │
      ▼
TASK_RUNNING
      │
      ▼
Scheduler
      │
      ▼
Task B 再次运行
      │
      ▼
重新尝试获得 mutex
```
